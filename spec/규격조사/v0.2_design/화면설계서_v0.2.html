현장촬영AI 규격조사 모듈
화면설계서 v0.2

v1.0(탄산화 조사) 앱에 규격조사 모듈을 추가하기 위한 화면설계서. 줄자 기반 폭·간격·홀 깊이 측정 3가지 subtype을 다루며, v1 디자인 톤(흰 배경 + 네이비 텍스트 + 파랑 액센트)을 유지한 채 측정 메타데이터 수집을 위한 마커·핸들·라디오 등 신규 컴포넌트를 추가했다. 9개의 mockup HTML로 구성되며, 각각은 데이터/인터랙션 상태와 기획 명세를 모두 포함한 완성형 산출물이다.

§1문서 개요

본 문서는 9개의 단독 mockup HTML과 그 명세를 통합한 마스터 화면설계서다. 각 mockup은 v1 실제 빌드(lib/pages/...)의 디자인 토큰을 1:1로 적용했고, v0.2 신규 컴포넌트는 그 디자인 톤을 자연스럽게 따라간다.

§4디자인 시스템

v1 코드(lib/pages/...)의 디자인 토큰을 그대로 사용. v0.2 신규 컬러는 측정 의미별로 분리.

주요 컴포넌트

  • 마커 (composite) — 측정 지점(point) + 라벨(label pill) + stem(SVG line) 3요소 분리. point/label 각각 자유 드래그, stem 자동 연결. (S-004 / S-007)
  • 가이드박스 핸들 — 32×32 흰 외곽 원, 가이드박스 외곽 18px에 위치. 측정점은 박스 좌/우 끝 얇은 세로선(measure-line)로 표시. (S-003)
  • 입력 필드 — type=text inputmode=decimal, 56px height, 20px monospace, 우측 × 클리어 버튼. (S-005 / S-008)
  • subtype 배지 — 보관함 카드의 dimension 레코드 표시. 폭/간격=파랑, 홀깊이=amber. (S-009)
  • 단계 안내 카드 — 번호 원 + 한 줄 제목. amber/blue 톤으로 측정 의미 구분, 완료 시 초록 ✓. (S-004 / S-007)

SurveyType 확장

SurveyType.dimension 신설 (v1의 rebarSpacing은 v2에서 dimension으로 대체).

SurveyRecord 확장

measurementSubtype (nullable)와 annotation (DimensionAnnotation?) 필드 추가. v1 탄산화 호환 유지.

DimensionAnnotation 신설

좌표·측정값·속성·이미지 메타를 묶는 클래스. 폭/간격은 start/endPointPixel + startValue/endValue 사용. 홀 깊이는 holeLipPixel + tapeReadingPixel + visibleReadingValue 사용.

S-005수치 입력 — 폭/간격 (공통)

S-005  수치 입력 — 폭/간격 ↗ 새 창으로 열기
  • type=text + inputmode=decimal (모바일 키패드 자동 + spinner 제거).
  • 폭은 시작값 보통 0, 간격은 임의 값 (예: 120). 측정값 = end - start.
  • 학습 품질 자가 체크 제거 — 비고 자유 기입으로 통합 (작업자 입력 부담 ↓).
  • 끝값 ≤ 시작값 → 빨간 에러 + 저장 비활성.
  • 저장 액션 2개 (v1 패턴) — "저장 후 계속 촬영"(primary, 카메라 S-003으로 복귀해 연속 측정), "저장 후 메인으로"(outlined, 메인 S-001 복귀). 좌측 "다시 촬영"은 미저장 폐기 + 카메라 복귀.

  • 데이터 상태 5종 표시: ① 빈 보관함 ② 데이터 있음 ③ 수정 모달 — 탄산화 ④ 수정 모달 — 폭/간격 ⑤ 수정 모달 — 홀 깊이
  • 탄산화 + 규격조사 3종(폭·간격·홀깊이) 카드 모두 포함 — 4개의 측정 유형이 한 화면에 어떻게 표시되는지 확인 가능.
  • dimension 레코드 카드에 subtype 배지 (폭/간격=파랑, 홀깊이=amber).
  • 썸네일 SVG도 측정 유형별로 차별화: 탄산화(균열), 폭(부재+줄자 0부터), 간격(두 부재+줄자 중간부터), 홀깊이(원형 홀+amber/파랑 마커).
  • 수정 모달 subtype별 분기 (v0.2 변경) — 카드의 data-survey-type에 따라 3가지 변형:
    • carbonation → 탄산화 모달 (단일 값 + 비고, v1 패턴)
    • dimension-width / dimension-gap → 폭/간격 모달 (시작값 + 끝값 + 자동 계산 + 비고, S-005 구조)
    • dimension-hole → 홀 깊이 모달 (줄자 읽은 값 + 비고, S-008 구조)
