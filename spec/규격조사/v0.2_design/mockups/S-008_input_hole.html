SCREEN MOCKUP & SPEC · S-008

수치 입력 — 홀 깊이

홀 입구에서 보이는 줄자 숫자(visible_reading_value) 단일 입력 → 깊이값으로 그대로 사용. 폭/간격과 달리 시작값-끝값 차이 계산이 아님.

💡 줄자 숫자 입력 = 깊이값 그대로 · 우측 × 빠른 삭제 · 0 이하면 에러 + 저장 비활성.

입력 검증 (실시간)

  • visible_reading_value · 숫자(소수 허용)여야 함. 빈 값이면 결과 카드 회색 + "—"
  • 0보다 커야 함 · 0 이하면 입력란 빨간 외곽선 + 에러 메시지 + 결과 카드 회색 + 저장 비활성
  • 깊이값 = 입력값 그대로 · 폭/간격(end - start)과 달리 단순 대입
  • 저장 버튼 활성 조건 · visible_reading_value 유효 + 0 초과. 그 외엔 회색 disabled
  • 비고는 선택 입력 — 저장 활성 조건에 영향 없음

  • 홀 깊이는 두 점 차이가 아님 · 줄자 끝이 바닥에 닿고, 홀 입구에서 보이는 숫자가 곧 깊이
  • 학습 데이터 가치 · 홀 입구 위치(hole_lip_pixel) + 줄자 읽기 위치(tape_reading_pixel) + 숫자(visible_reading_value) 세 요소가 함께 저장되어 AI가 "어디서 무엇을 읽었는지" 학습 가능
