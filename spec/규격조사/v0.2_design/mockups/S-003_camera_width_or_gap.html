폭 (wall_or_member_width) vs 간격 (member_gap) — 줄자 시작 위치 차이

두 subtype의 핵심 차이는 줄자의 어느 눈금을 측정 시작점에 맞추는지다. 측정 대상의 실제 사이즈 계산 결과는 같지만, 줄자 사용 케이스가 다르므로 학습 데이터로서 각각 의미가 있다. UI/UX는 동일하므로 같은 화면을 공유하되, S-002에서 선택된 subtype에 따라 안내 문구만 분기.

구분폭 (wall_or_member_width)간격 (member_gap)
줄자 시작줄자 0 = 측정 시작면줄자 임의 눈금 (예: 120) = 측정 시작점
안내 문구"측정 시작면에 맞추고 촬영하세요""줄자 양 끝의 눈금이 모두 보이게 촬영하세요"
시작값 입력 (S-005)일반적으로 0임의 값 (예: 120)
측정값 계산= end - 0 = end= end - start
현장 시나리오부재 끝에 줄자 0을 정확히 맞출 수 있을 때부재 모서리 손상·접근 어려움 등으로 줄자 0이 아닌 임의 눈금에서 시작해야 할 때
가이드박스/핸들/줌/플래시/셔터/썸네일완전 동일 (단일 mockup 공유)
  • 학습 데이터 가치 · 폭은 "줄자 0이 항상 측정 시작" 케이스, 간격은 "줄자 임의 위치에서 시작" 케이스를 분리 수집 → 모델이 줄자 0 의존 없이 학습 가능
  • 본 mockup 표시 상태 · 안내 버블에는 폭 케이스 문구("측정 시작면에 맞추고 촬영하세요") 표시. 실제 앱에선 subtype 인자에 따라 두 문구 중 하나가 노출

학습데이터 품질 수집 정책

본 앱의 본질은 AI 학습용 정확한 데이터 수집이다. 단, 측정 영역 픽셀 폭은 줌·거리·대상 크기에 따라 의미가 달라지는 간접 지표라 작업자 판단 근거로 부적합. 또한 수치 입력 단계의 quality_flags 자가 체크(6개 항목)도 작업자 입력 부담 대비 일관성이 낮다고 판단해 제거. 품질 신호는 다음 방식으로 수집한다.

  • 현장 입력 · 비고(자유 텍스트)로 이상사항 자유 기입 (S-005)
  • 후처리 라벨링 · 학습 큐레이터/검수자가 비고+이미지를 기반으로 quality_flags 채움
  • 자동 분석 · 가능 시 이미지 분석 모델로 quality_flags 자동 추출 (향후)
  • 카메라 셔터 잠금 · 자세(기울기 > 16°)에만 적용. 픽셀 폭/edge 인접 등은 잠그지 않음
  • annotation 모델 구조 유지 · DimensionAnnotation.qualityFlags 필드는 그대로 두어 후처리 단계에서 채울 수 있게 함
