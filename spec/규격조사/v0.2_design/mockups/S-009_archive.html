수정 모달 — subtype별 분기 v0.2 변경

v0.2부터 수정 모달은 카드의 data-survey-type에 따라 3가지 변형으로 분기. 각 변형은 원본 입력 화면(S-탄산화 / S-005 / S-008)의 구조와 일치하여 작업자에게 친숙한 입력 패턴 유지.

카드 subtype모달 (stage)입력 구조원본 입력 화면플로우 ID
carbonation stage 3 — 탄산화 단일 측정값(value, mm) + 비고 v1 데이터 입력 화면 S-009E
dimension-width / dimension-gap stage 4 — 폭 / 간격 줄자 시작값 + 줄자 끝값 + 자동 계산(measured_value = end - start) + 비고 S-005 (수치 입력) S-009Ed
dimension-hole stage 5 — 홀 깊이 줄자에서 읽은 깊이값(visible_reading_value, mm) + 비고 S-008 (수치 입력) S-009Eh
  • 표시 트리거 · 카드 우측 "수정" 버튼 탭. .record-card[data-survey-type] 값으로 적절한 모달 분기
  • 편집 가능 필드 · subtype별 입력 모두 + 비고(SurveyRecord.note). 폭/간격은 시작/끝값 분리 편집 가능 (v0.1 → v0.2 정책 변경)
  • 편집 불가 · 사진, 좌표(startPointPixel·endPointPixel·hole_lip_pixel·tape_reading_pixel), subtype, timestamp — 모두 보관함에서 변경 불가 (재촬영 필요)
  • 저장 동작 · StorageService.I.updateRecord(updated) 호출 → records.json 갱신 → 카드 표시 갱신. 폭/간격은 startValue/endValue/measuredValue 셋 다 갱신, 홀 깊이는 visibleReadingValue + measuredValue 갱신
  • 모달 backdrop · 다크 반투명 + blur · v1 디자인 일관 유지
  • 플로우 매핑 · 인터랙티브 플로우 프로토타입에서 3개 sub-screen으로 분리 노출. mockup stage 3/4/5에 각 변형 시각화. 실제 라우트는 /archive 동일
