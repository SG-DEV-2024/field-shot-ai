정밀 보정 모드 (S-003 → S-004 연계)

  • 진입 인자에 사전 좌표 포함 · S-003 핸들 좌표를 원본 픽셀로 환산하여 prefilledStart, prefilledEnd로 전달
  • 마커 초기 배치 = S-003 핸들 위치 그대로 · onInit에서 사전 좌표를 preview 좌표로 역환산해 화면에 표시. 사용자가 S-003에서 가이드박스를 어떻게 조정했든 그 위치가 S-004 마커 초기값이 됨 — 보정이 끊기지 않고 그대로 이어짐
  • mockup 좌표 일치 확인 · S-003 기본 --start-x:60px / --end-x:300px ↔ S-004 마커 초기 vx:60 / vx:300 동일 좌표계(360px 폭). 시뮬 photo의 부재 edge 선도 동일 위치에 그려 시각 연속성 보장
  • 인터랙티브 플로우 — 사전 좌표 전달 메커니즘 · S-003 셔터 탭 시 _embed.js가 .screen의 --start-x / --end-x 값을 읽어 postMessage({state:{startX, endX}})로 부모에 전송. 부모는 다음 iframe URL에 쿼리(?startX=80&endX=280)로 전달. S-004 inline script가 URLSearchParams로 읽어 pos 초기값 + edge SVG x1/x2 갱신. 사용자가 핸들로 박스 폭을 조정했다면 그 조정 그대로 S-004 마커 위치와 시뮬 photo edge 위치에 반영됨
  • 처음 찍기 모드 아님 · 이전 와이어프레임은 "1단계 시작점 터치 → 2단계 끝점 터치" 였으나, v0.2는 처음부터 두 마커가 사전 좌표 위치에 표시된 상태로 진입
  • 학습데이터 정확도 ↑ · 카메라 자세 잡으면서 미리 선택한 위치 → 보정 → 정지 이미지 정밀 확인 3단계 검증
  • 보정량 기록 (옵션) · 원본 사전 좌표와 최종 좌표 차이(delta_x)를 quality_flags에 저장 시 학습 품질 분석에 활용 가능

좌표 처리 (preview ↔ original)

  • preview 좌표 (UI) · 사진 표시 영역 내 px (예: 360×380 frame 기준)
  • 원본 좌표 (저장) · BoxFit.contain 환산 — 원본 이미지(예: 1440×1080) 픽셀 기준 [좌상단 0,0]
  • 환산식 · scale = min(prevW/origW, prevH/origH) · renderedW = origW * scale · offset = (prevSize - renderedSize) / 2 · 원본 = (preview - offset) / scale
  • 저장 위치 · DimensionAnnotation.startPointPixel: [x, y], endPointPixel: [x, y]
  • "다음" 시 처리 · 마커 preview 좌표 → 원본 좌표 환산 1회 → partial로 S-005 전달

학습데이터 품질 수집 정책

본 앱의 본질은 AI 학습용 정확한 데이터 수집이지만, 작업자에게 픽셀 폭 표시나 quality_flags 자가 체크를 강요하는 것은 판단 정확도와 입력 부담 양 측면에서 효과가 낮음. 다음 방식으로 정리.

  • 현장 입력 · 비고(자유 텍스트)로 이상사항 자유 기입 (S-005)
  • 후처리 라벨링 · 학습 큐레이터/검수자가 비고+이미지를 기반으로 quality_flags 채움
  • 자동 분석 · 가능 시 이미지 분석 모델로 quality_flags 자동 추출 (향후)
  • S-003/S-004 픽셀 폭 표시 제거 · 줌·거리에 따라 의미가 달라지는 간접 지표라 작업자 의사결정에 도움 안 됨. 특히 S-004에선 마커를 잘못된 위치로 유도할 위험
  • annotation 모델 구조 유지 · DimensionAnnotation.qualityFlags 필드는 후처리에서 채워질 수 있도록 그대로 유지
